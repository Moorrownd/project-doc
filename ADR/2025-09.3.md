# ADR: 2025-10.3 - Выбор технологии программирования клиентского приложения

Дата: 28 октября 2025 г.

Статус: Сдано

## Контекст

Клиентская часть системы предназначена для взаимодействия пользователей (владельцев парковочных мест, арендаторов и операторов) с серверной частью: поиск парковок, бронирование, оплата и управление данными.

## Рассмотренные варианты

- JavaScript / TypeScript для веб-клиента.
- TypeScript для кроссплатформенного мобильного клиента.
- Нативные языки мобильных платформ.

## Решение

Для клиентской части выбран язык программирования TypeScript.

## Обоснование

- Использование одного языка для веб- и мобильного клиента снижает сложность разработки и сопровождения.
- Статическая типизация повышает надёжность клиентского кода.
- Язык хорошо подходит для работы с REST API и форматом JSON.

## Недостатки решения

- Требуется этап сборки и транспиляции.
- Зависимость от экосистемы JavaScript.

## Последствия

- Унификация клиентской разработки.
- Упрощение поддержки и расширения клиентских приложений.
- Повышение качества клиентского кода.
