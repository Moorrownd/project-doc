# ADR: 2025-09.4 - Выбор системы хранения данных

Дата: 25 декабря 2025 г.

Статус: Сдано

## Контекст

Система требует хранения данных о пользователях, парковочных местах, бронированиях и их статусах. Хранилище данных должно быть согласовано с серверной технологией.

## Рассмотренные варианты

- Реляционная база данных MySQL.
- Альтернативные СУБД.

## Решение

Выбрана **реляционная база данных MySQL**.

## Обоснование

- Использование MySQL соответствует серверной части на PHP Symfony.
- Реляционная модель хорошо подходит для описанных сущностей и связей UML.
- Поддержка транзакций важна для операций бронирования.

## Недостатки решения

- Требуется проектирование схемы и миграций.
- Масштабирование требует дополнительной настройки.

## Последствия

- Обеспечивается целостность и согласованность данных.
- Упрощается реализация запросов и отчётов.
- База данных легко интегрируется с ORM.
