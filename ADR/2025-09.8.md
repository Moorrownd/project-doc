# ADR: 2025-10.8 - Выбор формата и протокола обмена сообщениями между клиентом и сервером

Дата: 25 декабря 2025 г.

Статус: Сдано

## Контекст

Клиентские приложения взаимодействуют с серверной частью системы для поиска парковок, бронирования и оплаты. Необходимо выбрать единый формат и протокол обмена данными.

## Рассмотренные варианты

- SOAP.
- REST поверх HTTP.

## Решение

Выбран **REST API с использованием HTTP/HTTPS и формата JSON**.

## Обоснование

- JSON является стандартным форматом для веб- и мобильных клиентов.
- Простота отладки и расширения API.

## Недостатки решения

- Отсутствие строгой типизации сообщений.
- Возможность дублирования данных в ответах.

## Последствия

- Упрощается интеграция клиентских приложений.
- Повышается читаемость и прозрачность взаимодействия.
- Обеспечивается совместимость с внешними сервисами.
